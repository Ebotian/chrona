**设计要素映射**
- **标题与摘要**: 标题显著展示，卡片内显示短摘（由 `w(a)` 决定长度），点击展开完整 `body`。
- **时间信息**: 将 `t_pub` 作为主排序/视觉锚点，`t_mod` 用作次要元数据（显示“更新于”）。
- **可见性指示**: 对 `v(a)=1` 条目显示明显锁定/亲密徽章与占位预览，提示登录或授权。
- **检索高亮**: 搜索结果中高亮 `τ`、`β` 匹配子串并展示上下文片段（snippet）。
- **元数据条**: 在文章卡片/详情中放 `t_pub`、`t_mod`、`w`（读时估计）、标签/类别与唯一 ID（用于稳定引用）。
- **信息密度与排版**: 根据 `w(a)` 动态调整卡片高度与摘要字数；移动端更紧凑，桌面可用卡片网格或列表切换。
- **友链可视化**: 基于 `G_L` 提供两种视图：扁平列表（常规友链）与小型交互图（探索连接与互链），可用作侧栏或专页。
- **过滤与分面导航**: 暴露按日期区间 `D`、关键词 `τ`/`β`、可见性 `v` 的筛选控件；保留排序切换（按 `t_pub`/相关性/更新）。
- **可访问性与状态**: 显示结果总数、加载状态、空结果提示，并保持 URL 可分享（包含筛选/分页状态）。

**分页策略建议**
- **默认（主列表）— 游标分页（Cursor）**: 按 `t_pub` 降序使用基于时间或 `id` 的游标（page size 8–12）。优点：插入/更新时稳定、不易重复；适合无限滚动或“加载更多”交互。
- **归档/日期跳转— 基于日期的分页**: 按年/月分组（`t_pub`）的静态归档页，支持快速跳到某月/年，便于 SEO 与用户定位历史内容。
- **搜索结果— 页码分页（Offset+Page）或受限游标**: 因用户需要跳转到某页，显示页码与当前范围（如 1–10），同时在大量结果时后端用游标优化性能并返回总数估算。
- **混合策略**: 首页与推荐流用游标（更流畅），归档与搜索提供页码与“跳转到日期”功能（更可控与可分享）。
- **实现细节/UX**: 保持分页状态在 URL（可分享），打开文章后返回保留滚动位置；显示“显示 n / 总 m 条”的反馈；为移动端提供“加载更多”按钮而非自动无限滚动以节省带宽。
- **性能与稳定性**: 对大集合优先使用游标查询并限制最大偏移；对小/静态集合（归档）使用静态页或缓存的页码视图。

## 路由隔离与私密入口设计（/private）

目的：为“亲密访客”提供分层、更丰富的阅读体验，同时对普通访客保持无差别的公开页体验；入口采用弱显式路由（例如 `/private`）以便分享与记忆。

- 路由与体验概述：访问 `/private` 时页面可切换主题配色、展开更长摘要或直接显示附加内容；对已授权用户显示完整体验，对未授权用户呈现扩展预览与温和 CTA（提示登录/获取访问）。
- 实现选项（从弱到强）：
	- 纯前端路由主题切换（最简单）：仅通过前端判断路径加载不同样式与更长摘录（注意不要把敏感正文仅靠前端隐藏）。
	- 软授权（推荐）：前端优先展示扩展内容给已登录/有 token 的用户，未登录用户看到扩展预览并提示低摩擦授权路径（魔法链接、一次性 token）。
	- 共享魔法链接：生成短期有效的带 token 的分享链接，可在外部私下分享并控制有效期与访问次数。
	- 服务器端验证（最严格）：对真正敏感内容，服务器在 `/private` 路径进行权限判断并返回受保护的正文。
- 非侵入 UX 原则：
	- 不在公共列表广泛显示“锁”或“仅亲密可见”的醒目标签，避免普通访客产生排斥感。
	- 同一条目采用渐进式展示：公开页显示统一标题与短摘，`/private` 或授权后展开更长摘或全文。
	- 仅对已授权用户使用微妙的视觉差（轻微高亮、色带、小徽章），传达“属于你”的专属感但不张扬。
- SEO / 抓取注意事项：
	- 对 `/private` 路径的 Open Graph、RSS、sitemap 输出进行过滤或使用通用摘要，避免私密正文被搜索引擎或社交卡片抓取。
	- 必要时在 `/private` 页面头部加入 `noindex` 或使用 canonical 指向公开版本。
- 安全与缓存：
	- 切勿仅靠前端隐藏敏感数据；对敏感内容使用服务器权限或受限 token。
	- CDN/缓存键需区分 `/private` 与公开路径，避免将私密内容缓存至公共层。
- 分享与回退：
	- 分享优先使用带短期 token 的链接而非直接暴露静态私密 URL。
	- 打开文章并返回列表时保留滚动位置与过滤状态，提升可用性。

推荐默认策略：采用软授权 + 渐进式内容暴露。把 `/private` 作为轻量入口，默认不强制认证，但对未授权用户仅显示扩展预览并提供魔法链接获取完整访问；对真正敏感内容再使用服务器端验证。

示例短流程：
1. 用户打开 `/private`（或从个人链接进入）。
2. 前端检查登录/本地 token：若已授权，加载扩展样式并显示完整内容；若未授权，显示扩展预览并提供“获取访问”按钮。
3. 用户选择“通过魔法链接授权”：系统发送一次性短期 token，用户点击后回到 `/private` 并获得完整视图（token 存储短期 cookie）。
4. 分享时优先生成带 token 的短链；如需长期私密访问，则使用明确的服务器端鉴权。
